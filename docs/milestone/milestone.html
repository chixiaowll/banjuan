🤖 AI Assistant Overhaul · Background Export · Faster Open
The Claude plugin grew from "read my notes" into a research assistant that reads full text and edits content; export moved fully into a background process (no more blocking); built-in plugins became global (~/.banjuan/plugins); and opening a library was made systematically faster — no more reading file contents to hash, and the SQLite index now persists instead of being fully rebuilt on every open.
read_document EPUB text web search global plugins background export live tool trace export image fixes faster open persistent index 5200-file cap Intel build missing-file handling
  • Claude reads document text: PDF / EPUB (by page/chapter), txt / md / html, plus notes & annotations
  • New MCP tools: mindmap add/edit/delete nodes, tags, folders/move, deletes (with confirmation)
  • Web research (WebSearch / WebFetch): look things up, then write them into notes
  • Rich context: current library, open documents/notes, current page + its text, selected text
  • Live tool trace, kept after completion: thinking and each tool call's input/result
  • Global plugin dir (~/.banjuan/plugins): auto-installed on launch, shared across libraries
  • Export fully backgrounded: batch + single unified, never blocks browsing, accumulating progress panel (incl. PNG/SVG)
  • Fixes: handwriting export keeps images, embedded-mindmap export, image import on Markdown import
  • CLI note create takes --content / --file / stdin; independent note type filter; resizable AI panel
  • Faster open: import no longer reads file contents (id/hash derived from path), so import cost is independent of file size
  • Persistent SQLite index: version-gated, no longer fully rebuilt on every open — near-instant when nothing changed
  • Note/document links maintained incrementally on write; missing-file detection reads the DB, so opens do ~zero file reads
  • Physically deleted files are flagged "missing" (kept, recoverable): list badge + detail banner; added a "Rebuild Library" maintenance action
  • Hard 5200-file cap per library + a pre-check at directory selection, so picking a huge folder can't hang the app
  • macOS now builds both Apple Silicon (arm64) and Intel (x64)
